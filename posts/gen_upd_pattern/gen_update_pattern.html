D3js General Update Pattern

The visualization on this page is an adaptation of Mike Bostock's tutorial on the d3js General Update Pattern[1]. In dynamic systems where data is often, frequently, or constantly changing the visualization must keep track of which data is new, which data remains unchanged, and which data is obsolete and must be removed. D3js excells at this kind of data binding and updating.

This is a visualization using code that I adapted a few years ago. The D3 update pattern has been re-tooled to make it easier to use and understand. Studying the latest version and applying it is now at the top of my do list.

[1] Mike Bostock. “General Update Pattern, III” (deprecated). Updated August 5, 2020
